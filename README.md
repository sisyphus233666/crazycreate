# 💩 拉了么

（本地文件夹或口头有时也会叫 **「拉屎项目」**，指的是本仓库。）

**纯前端、零依赖** 的「今天拉了没」记录器：记次数、记时间，数据只躺在你的 **浏览器 `localStorage`** 里——**不上传、不登录**。手机用着顺了，可以 **添加到主屏幕**，当个轻量 PWA 用。

---

## 能做什么

| 能力 | 说明 |
|------|------|
| **多事项** | 顶部可切换不同记录项；**「我的事项」**里可新增，自定义**名称**、**表情符号**（今日大图）、**日历格图标**（内置图标库任选）。默认仍为「拉了么」一条。 |
| **今日** | 主按钮记一笔；文案、表情、角标随次数变化；列表支持长按多选删除。 |
| **回顾** | 月历按**当前事项**显示；有记录的日子显示该事项选的 SVG 标记；点某一天看详情；**当月首次进入会默认展开今天**。 |
| **补记** | 便便事项仍显示「补拉 / 💩时间」；其它事项为「补记 / 时间」。 |
| **删除** | 今日 / 回顾详情里长按进入多选，底部操作条删除。 |

## 名字怎么分工（避免困惑）

- **主屏幕 / 安装名**：**「拉了么」**（`manifest.json` 的 `name` / `short_name`，以及 `apple-mobile-web-app-title`）。
- **浏览器标签 & 页面里 0 次主文案**：**「今日拉了吗」**（`<title>` 与 `todayHeadUI`）。

## 数据存在哪

- 主键：**`localStorage['poop-tracker-data']`**。
- **当前格式（v2）**：内含 `trackers`（事项列表：名称、`emoji`、`iconId`）、`activeTrackerId`、`records`（按事项 id → 按日期 → `{ times, count }`）。首次打开会自动把 **旧版单层日期字典** 迁成默认事项 **`trk-poop`（拉了么）** 的数据。
- **一次性兼容**：若本机曾经有过「分档案」试用留下的 `poop-tracker-data:*` 等数据，启动时会 **尽量合并回主键**（仅当主键尚空时），详见 `index.html` 内 `foldInProfileDataOnce`。

**没有云同步**；换机或清站点数据会丢记录，重要的话请自担备份策略。

## 部署到 GitHub Pages（提要）

1. 把本仓库推到 GitHub，**Settings → Pages** 选发布分支与目录。  
2. **`manifest.json`** 已使用 **`"start_url": "./"`**、**`"scope": "./"`**，适合 **`https://<用户>.github.io/<仓库名>/`** 这类子路径。  
3. 推送后等一两分钟再访问；更新后若图标/名字不刷新，可删了快捷方式再加，或递增 `sw.js` 里的 **`CACHE_NAME`**（当前为 **`poop-tracker-v8`**）触发缓存刷新。

日常若在别的文件夹（如 **`poop-tracker`**）改代码、真实 Git 仓库在 **`poopoo`**：把文件 **复制进 clone 目录** 后在后者里 **`git commit` + `git push`**——**[`docs/plans/2026-04-29-poop-tracker-design.md`](docs/plans/2026-04-29-poop-tracker-design.md)** 第六章写得更细。
